Notable stuff

Polyfill.io

I redefined how polyfills are tested for polyfill.io.
Originally polyfills were tested only in isolation, which led to various bugs when polyfills interacted.
I also fixed most of the bugs that were surfaced by these new tests.

I contributed various polyfills to polyfill.io and still help to maintain it.

core-web

I maintain core-web, which is the same set of polyfills from polyfill.io, but injected through babel.

Browser Compat

I am a regular contributor to MDN Browser Compat Data..
I also created web-tests as a way to quickly gather historical support data for various web features.

I am critical of MDN/Google Baseline and will keep advocating for something better and less biased.
I maintain an alternative (mdn-support-context) to illustrate that something better is technically possible.

CSS

PostCSS Preset Env

I took over maintenance for PostCSS Preset Env (together with Antonio Laguna).

  • migrated everything to a monorepo
  • resolved all open issues and bug reports
  • massively increased test coverage
  • automated the release process

CSS Color

I created implementations for css-color-4 and css-color-5.

  • worked together with the editors to improve these specifications
  • contributed various new tests and fixes to WPT
  • released these tools as PostCSS plugins
  • still advocating for browser vendors to implement these features correctly

CSS Tokenizer and Parsers

I created a correct CSS tokenizer and various parsers.

  • CSS tokenizer
  • re-usable tests for CSS tokenizers
  • CSS parser algorithms
  • media query parser
These tools drive much of the new PostCSS plugins and are also used in projects like Stylelint.
